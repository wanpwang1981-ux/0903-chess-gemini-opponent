# Bug 修復歷程

本文檔記錄了在中國暗棋遊戲開發過程中發現的主要 Bug 及其修復方案。

## Bug 1: 遊戲在第一步翻棋後提前結束

- **發現時間:** 2025-09-07
- **問題描述:** 在任何對戰模式下，當玩家翻開第一枚棋子後，系統會立即顯示「遊戲結束！XX方獲勝！(對方已無棋子)」，導致遊戲無法繼續。
- **根本原因:** `chess.js` 中的 `checkGameOver` 函式在檢查遊戲是否結束時，包含了「檢查玩家**已翻開**的棋子數量」的邏輯。當第一枚棋子被翻開時，對手方已翻開的棋子數為 0，這錯誤地觸發了勝利條件。
- **修復方案:**
    - 編輯 `chess.js` 中的 `checkGameOver` 函式。
    - 移除了僅基於已翻開棋子數量的判斷邏輯。
    - 將遊戲結束的判斷完全交由 `hasAnyValidMoves` 函式處理。該函式會正確地檢查玩家是否還有任何合法的移動（包括移動自己的棋子或翻開任何暗棋），確保遊戲只有在玩家確實無棋可走時才會結束。
- **狀態:** 已修復。

## Bug 2: 電腦 (AI) 移動不遵守遊戲規則

- **發現時間:** 2025-09-07
- **問題描述:** 在「對戰電腦」模式下，AI 會執行不符合暗棋規則的移動（例如，錯誤的吃子或移動到非相鄰的格子）。
- **根本原因:** 為了讓 `ai.js` 模組獨立，開發初期從 `chess.js` **複製**了一份規則判斷函式（如 `getAvailableMoves`, `canCapture` 等）到 `ai.js` 中。這種程式碼重複導致了兩個問題：
    1.  維護困難：當 `chess.js` 中的主規則更新或修復時，`ai.js` 中的副本可能被遺忘，導致規則不一致。
    2.  潛在錯誤：複製的程式碼本身可能就存在細微的錯誤。
- **修復方案 (進行中):**
    1.  **重構 `chess.js`:** 將 `chess.js` 中的核心規則函式（如 `getAvailableMoves`）修改為更通用的形式，使其可以接收 `board` 狀態作為參數，而不是依賴全域變數。
    2.  **重構 `ai.js`:** 刪除 `ai.js` 中所有重複的規則判斷函式。
    3.  **統一呼叫:** 讓 `ai.js` 直接呼叫 `chess.js` 中唯一的、權威的規則函式來產生移動。
    *   此方案旨在消除程式碼重複，確保 AI 和人類玩家遵循完全相同的規則，從根本上解決問題。
- **狀態:** 正在修復中。
